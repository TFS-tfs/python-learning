我用Python给你做了一个小盗贼

我有个疯狂的想法，重写《Learn Python the Hard Way》第三部分，用Rogue来教面向对象编程。为了测试这个想法，我用 Python 创建了一个基于诅咒的简单 Rogue 实现。

什么是《叛变》？

《叛变》是80年代极具影响力的游戏，完全在终端中通过文本字符如和进行渲染。它拥有许多独特的特色，结果发现非常有趣，这也是为什么你今天能在许多被称为“Roguelike”的游戏中看到这些特色。《盗贼》的一个关键特色是，当你死亡时，物品会保留在你的“工具腰带”上。这让下一次副本通关稍微容易一些，让你在再次死亡前获得更好的装备，重复这个过程。再加上随机敌人和关卡，你就拥有了一款无限可玩的游戏。@#

《叛变》还有一点有趣的是，画面的缺失似乎并不重要。人们很快就能忽略文字字符的使用，直接理解一切含义，无需大量视觉反馈。我甚至会说，《叛变》证明了你不需要华丽的画面和色彩也能做出一款伟大的游戏。

这也是为什么Rogue是许多程序员常见的“有趣项目”。Rogue完全由程序员设计，因为它都是算法创造机制。这是因为游戏中许多内容被简化了。需要碰撞吗？只要看看广场上有没有什么东西。需要战斗吗？玩家是否进入了有人占据的格子？需要路径规划吗？只需用迪克斯特拉算法，你就拥有了所有需要的。没有图形需要担心，没有实时动态，没有时间间流畅的 LERP，只有纯代码。

我也觉得几乎任何人都能做《叛变》游戏。你能搞懂诅咒库吗？你能做一个二维数组吗？那这大概就是你需要的。没有游戏引擎、3D图形、线性代数、复杂三角函数，只有你、一个网格和时间。你肯定你的游戏

为什么？

我之前说过Rogue是个很棒的初学者项目，但我其实没测试过。我大概知道它是基于我用C++轻松创建的，但我实际上并不是专门为了教学而做的。这通常和我自己开发的制作过程不同。为了解决这个问题，我创建了“诅咒你”Python Rogue项目，确保这真的是初学者能完成的。

整个实现过程大约花了我4小时，加上另外4到8小时重写和整理。重写中最大的变化是网格数据和作改用numpy。Numpy是游戏中作网格最简单的方式，也是学习Numpy的好机会。该实现包含：

  1. 一个猎杀迷宫生成器，将房间“打孔”进迷宫以增加变化。
  2. 一种基于迪克斯特拉算法的路径规划算法，用于引导敌人到玩家。
  3. 一个碰撞系统......碰撞。
  4. 一个战斗系统。
  5. 一个敌人，还有一种随机生成任意数量敌人的方式。

整个文件大约包含350行Python，我从只有82行Python的起点分阶段构建。

基于这些，我觉得学过Python基础的人可能能把它当作一个项目来做。这需要一个好的引导者，但这正是我课程的重点。

那面向对象呢？

目前我教面向对象编程的方式是教学生如何用文字和结尾来构建自己的面向对象编程。这解决了面向对象编程被“重言式”教授的常见问题。如果你读过OOP的描述，你就明白我的意思：

面向对象编程是一种编程风格，将代码结构化为对象和类，这些对象也是对象，并且有方法位于对象上。哦，方法就是对象。很简单！

通过展示面向对象编程不过是指令和闭合（加上一堆语法糖），你就能绕开同义反复的解释，人们真正明白发生了什么。就像我现在用 和 教大家自己做循环一样。gotoif-statements

虽然这种方法有效，但存在一个问题：为什么要这样做，甚至是否有用，仍然不明显。这个小小的Rogue项目的想法是给大家一个具体的例子，展示如何用面向对象编程来制作真实的东西，并展示他们如何评估何时以及为什么使用面向对象来制作东西。

怎么做到的？

我从简单的初版实现了《诅咒你》Python Rogue的初稿。如果你从到，你会得到一个基础的盗贼，包含战斗、碰撞、迷宫生成和敌人。不过我不太确定我是否喜欢这种做法，因为它更多是重构练习，而不是如何制作这种类型的游戏。另一方面，这也展示了如何将一段非面向对象编程的代码转化为面向对象编程。phase_01.pyphase_13.py
